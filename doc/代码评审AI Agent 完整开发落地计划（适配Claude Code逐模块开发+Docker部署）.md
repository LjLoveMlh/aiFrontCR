# 代码评审AI Agent 完整开发落地计划（适配Claude Code逐模块开发\+Docker部署）

## 一、项目基础信息

### 1\. 项目背景

日常代码提交频繁被CodeReview指出代码不规范、不符合团队编码习惯、重复历史踩坑问题；自研私有化代码评审Agent，提交代码前自动预检，完全贴合团队规则与历史评审习惯，彻底规避CR重复问题。

### 2\. 核心依赖与对标学习教程

- 学习参考：[AI智能体实战速成指南（LangChain\+LangGraph企业级Agent教程）](https://didilili.github.io/ai-agents-from-zero/#/README)

- 开发工具：Claude Code（全程AI辅助逐行编码、调试、排错）

- 核心框架：LangChain（主流程）\+ LangGraph（结构化评审工作流），向量栈**完全对齐教程18节Redis向量库\+Embedding实战**，全套生产级RAG检索方案；大模型全线切换阿里通义千问，适配国内网络环境，无网络访问障碍

- 大模型：**阿里通义千问Qwen3\-Max**（国内直连无翻墙、接口稳定低延迟、中文\+前端代码理解能力极强；原生兼容LangChain/LangGraph、支持结构化输出、Function Calling，完美适配代码评审Agent，彻底解决Claude国内网络访问卡顿、密钥风控、需要魔法网络的痛点）

- 知识库原料：团队CLAUDE\.MD编码规范、历史全部CodeReview评审记录

- 部署方式：全程Docker容器化部署，环境统一、开箱即用

- 触发入口：CloudCode编辑器手动调用 \+ Git pre\-commit提交自动拦截

### 3\. 项目核心能力（最终交付效果）

1. 双知识库RAG检索：自动匹配团队编码规范\+历史同类CR错误，拒绝通用大模型无效评审

2. 代码差异化评审：仅校验本次git变更代码，不全局扫描，速度更快

3. 分级评审结果：阻断性问题（必须修改，阻止提交）\+ 优化建议（可选修改）

4. 知识库增量更新：新增CodeReview记录无需重构全量向量库，一键增量入库

5. 本地无感运行：编辑器一键调用\+git提交自动触发，贴合日常开发流程

6. 容器化交付：Docker一键启动，无需配置本地Python环境

## 二、整体技术架构（对标教程企业级RAG\+LangGraph工作流）

## 六、完整生产级技术栈最终选型（前端CR专项 \+ 贴合教程 \+ 简历亮点）

### 1、大模型选型变更说明（核心替换：Claude 3\.5 → 通义千问Qwen3\-Max）

- **最终大模型**：阿里通义千问 Qwen3\-Max

- **替换原因（解决你的核心痛点）**：Claude海外模型国内必须魔法网络，接口不稳定、时常超时、密钥风控严格、响应延迟极高；通义千问国内阿里云官方节点直连，无需翻墙、7x24稳定、低延迟、个人API额度充足、调用成本极低

- **代码评审专项优势**：原生中文理解拉满，完美贴合团队中文CR评审话术；对前端JS/TS/Vue/React代码纠错、规范检测能力优秀；32K长上下文无损读取完整规范文档与代码片段；原生兼容LangGraph状态机、结构化输出，无需额外Prompt适配

- **简历面试加分点**：贴合国内企业AI项目真实技术选型，国内生产项目90%以上均使用阿里千问/文心一言等国产大模型，比海外Claude更贴合国内面试与工程实际场景
